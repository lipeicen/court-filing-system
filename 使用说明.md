# 法院保全案件管理系统 - 使用说明

## 一、系统登录

1. 打开浏览器，访问：`http://localhost:5000`
2. 输入管理后台账号密码登录
3. 系统会记住当前登录账号，后续案件导入、自动立案都按该账号执行

> 注意：管理后台账号存在 `system_users` 表，和法院系统登录账号是两套账号。

---

## 二、配置法院系统账号

1. 登录管理后台后，点击右上角用户名下拉或左侧菜单中的 **"系统配置"**
2. 输入法院系统登录账号、密码、身份
3. 点击 **保存配置**

配置规则：
- 每个管理后台用户可独立配置一套法院账号
- 配置保存在 `system_config` 表，key 为 `login_username_手机号`、`login_password_手机号`、`login_user_type_手机号`
- 登录后台时也会自动同步当前账号到 `system_config`

---

## 三、批量导入案件

### 1. 准备模板

模板文件：`templates/案件批量导入模板.xlsx`

模板包含 73 列，主要字段：
- 案件信息：`case_no`、`case_name`、`court_name`、`preserve_amount`、`preserve_category`
- 申请人信息：`applicant_type`、`applicant_name`、`applicant_cert_no`、`applicant_phone`
- 被申请人信息：`respondent_type`、`respondent_name`、`respondent_cert_no`、`respondent_phone`
- 财产线索：`property_type`、`bank_name`、`bank_account`、`amount`
- 担保信息：`guarantee_type`、`guarantee_amount`、`guarantor_name`
- 代理人信息：`agent_name`、`agent_phone`、`law_firm_name`

### 2. 导入操作

1. 在案件列表页点击 **"批量导入"**
2. 选择填写好的 Excel 文件
3. 点击上传

导入规则：
- 任何一条数据校验失败，整个导入都会回滚，不会写入部分数据
- 案件 `case_no` 已存在时，导入失败
- 导入成功后，案件 `created_by` 记录为当前登录管理后台账号

### 3. 常见错误

| 错误信息 | 原因 | 处理 |
|---|---|---|
| 缺少必填字段 applicant_cert_no | 申请人身份证号为空 | 补全数据后重新导入 |
| 缺少必填字段 respondent_id | 被申请人身份证号为空 | 补全数据后重新导入 |
| 案件编号已存在 | 该 case_no 已导入过 | 删除旧案件或更换 case_no |

## 四、自动立案

### 1. 单个案件自动立案

```bash
python final_auto_upload_db_v3.py 保全2027001
```

### 2. 批量自动立案

运行自动立案脚本，系统会自动处理所有未提交的案件：

```bash
python final_auto_upload_db_v3.py
```

（不传 case_no 时使用默认案件，建议传具体案件号）

### 3. 自动立案流程

1. 读取案件数据
2. 根据 `cases.created_by` 从 `system_config` 读取对应法院账号
3. 启动浏览器，打开法院保全网站
4. 自动登录（OCR 识别验证码）
5. 依次填写：申请人、被申请人、财产线索、担保信息、上传材料
6. 提交保全申请

### 4. 自动登录测试

```bash
python auto_login.py
```

默认使用 `.env` 中的账号，也可以传入创建者账号：

```bash
python auto_login.py 13723715831
```

## 五、案件状态同步

手动同步：

```bash
python sync_filing_status.py
```

或运行批处理：

```bash
run_sync.bat
```

同步会抓取法院网站的案件状态，更新到 `cases.filing_status` 字段。

## 六、多用户账号说明

场景：
- 用户 a 登录管理后台，配置法院账号 A，导入案件用 A 自动立案
- 用户 b 登录管理后台，配置法院账号 B，导入案件用 B 自动立案

实现方式：
- `system_users` 存管理后台登录账号
- `system_config` 按用户存多套法院账号
- `cases.created_by` 记录案件属于哪个管理后台用户
- 自动立案时根据 `created_by` 取对应法院账号

## 八、常见问题

1. **导入失败但部分数据已写入**
   - 已修复为事务性导入，失败会回滚

2. **自动立案用错账号**
   - 检查 `系统配置` 是否配置正确
   - 检查案件 `created_by` 是否对应当前登录用户

3. **验证码识别失败**
   - 使用本地 OCR `ddddocr`
   - 确保验证码图片清晰

4. **浏览器看不见操作过程**
   - 当前 `final_auto_upload_db_v3.py` 已设置为 `headless=False`

5. **案件删除失败，外键约束错误**
   - 先删除该案件下的 `property_clues` 记录，再删除案件

## 九、数据库关键表

| 表名 | 用途 |
|---|---|
| `users` | 旧管理后台用户表 |
| `system_users` | 当前管理后台用户表 |
| `system_config` | 系统配置，按用户存法院账号 |
| `cases` | 案件主表，含 `created_by` |
| `property_clues` | 财产线索 |
| `case_files` | 案件材料 |
| `agents` | 代理人信息 |
